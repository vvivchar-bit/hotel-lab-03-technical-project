## Технічний проект програмної системи «Готель»

## Тема роботи

Технічний проект.

## Мета роботи

Набути навичок з розробки технічного проекту програмного забезпечення.

## Програмна система

**Програмна система «Готель»** — це навчальний вебзастосунок для автоматизації основних процесів роботи готелю: перегляду номерів, пошуку доступних варіантів, створення бронювань, збереження даних клієнтів та контролю стану номерів.

Система орієнтована на використання клієнтом та адміністратором. Клієнт може переглядати доступні номери та створювати бронювання, а адміністратор може контролювати список бронювань і змінювати статуси номерів.

Технічний проект програмної системи «Готель» визначає основні технічні рішення, середовище розробки, структуру програмного забезпечення, основні компоненти системи, їх взаємодію та засоби реалізації.

## 3.1 Вибір середовища програмування

Для розробки програмної системи «Готель» було обрано середовище **Visual Studio Code**.

Visual Studio Code є зручним редактором коду, який підтримує роботу з HTML, CSS та JavaScript, має підсвічування синтаксису, вбудовану роботу з Git, підтримку розширень і дозволяє швидко створювати та редагувати вебзастосунки.

Для реалізації програмної системи використано такі технології:

- **HTML** — для створення структури вебсторінки;
- **CSS** — для оформлення інтерфейсу користувача;
- **JavaScript** — для реалізації логіки роботи системи;
- **LocalStorage** — для збереження даних у браузері;
- **GitHub** — для зберігання файлів проекту та результатів роботи;
- **PlantUML** — для створення UML-діаграм.

Обраний стек технологій є достатнім для створення навчального клієнтського вебзастосунку, який працює без серверної частини та бази даних.

---

## 3.2 Опис структури програмної системи

Програмна система «Готель» складається з трьох основних частин:

1. **Інтерфейс користувача** — відображає сторінку системи, список номерів, форму пошуку та форму бронювання.
2. **Логіка роботи системи** — обробляє дії користувача, створює бронювання, змінює статуси номерів і оновлює статистику.
3. **Сховище даних** — забезпечує збереження інформації про бронювання за допомогою LocalStorage.

Основні функції системи:

- перегляд статистики готелю;
- перегляд списку номерів;
- пошук номерів за типом або статусом;
- створення бронювання;
- збереження даних клієнта;
- перегляд списку бронювань;
- зміна стану номерів.

## 3.7 Опис технічних засобів та програмного забезпечення

Для роботи програмної системи потрібні такі програмні засоби:

- веббраузер Google Chrome, Microsoft Edge або інший сучасний браузер;
- редактор коду Visual Studio Code;
- система контролю версій Git;
- платформа GitHub;
- онлайн-редактор PlantUML або PlantText для побудови діаграм.

Мінімальні технічні вимоги:

- персональний комп’ютер або ноутбук;
- операційна система Windows, Linux або macOS;
- доступ до веббраузера;
- доступ до інтернету для роботи з GitHub;
- мінімум 4 ГБ оперативної пам’яті.

Оскільки система реалізована як клієнтський вебзастосунок, вона не потребує окремого сервера або встановлення СУБД.

## 3.8 Проектна оцінка надійності системи

Надійність програмної системи «Готель» забезпечується простотою архітектури, використанням клієнтських вебтехнологій та збереженням даних у LocalStorage.

Переваги системи з точки зору надійності:

- система не залежить від зовнішньої бази даних;
- основні файли легко перевіряти та редагувати;
- дані зберігаються локально у браузері;
- інтерфейс працює без складного серверного налаштування;
- структура проекту є простою та зрозумілою.

Можливі слабкі місця:

- дані LocalStorage можуть бути очищені користувачем;
- система не має серверної авторизації;
- при великій кількості даних LocalStorage не є повноцінною заміною бази даних;
- система залежить від коректної роботи браузера.

Для навчального проекту обрана архітектура є достатньою, оскільки дозволяє продемонструвати основні функції системи без ускладнення реалізації.
